# スクリーンショット撮影手順

Chrome Web Store の掲載には、**1280x800 または 640x400** のスクリーンショットを **1〜5 枚** 添付できます。1280x800 推奨。

撮影画像は `docs/screenshots/` に配置 (Git 管理してOK、Web Store にはダッシュボードからアップロード)。

---

## 推奨カット (5 枚)

### 1. ポップアップで期間選択中の状態
**キャプチャ対象**: YouTube チャンネルページを背景に、拡張ポップアップを開いている画面
- ポップアップの「チャンネル」欄に @handle が自動入力されている
- 「期間」プルダウンを開いて選択肢が見える瞬間
- **キャッチコピー画像で使う訴求点**: 「ワンクリックで期間 × 人気順を絞り込み」

### 2. 結果ページ (人気上位)
**キャプチャ対象**: 結果ページのトップ部分。サムネイルが整然と並んでいるところ
- ヘッダーで「過去 1 か月」「100 件」「再生回数」が選ばれている状態
- 上位 6〜9 件のカードが表示されている
- **訴求点**: 「期間別の人気動画が一目瞭然」

### 3. 並び順切替 (高評価順 など)
**キャプチャ対象**: 「並び順」セレクターを「高評価」に切り替えた直後
- ランキングが #1, #2... と見えている
- **訴求点**: 「再生回数 / 高評価 / コメント数 で並び替え」

### 4. キャッシュ表示 + 強制再取得
**キャプチャ対象**: ヘッダーに「キャッシュ ・ 2 時間前」バッジが表示された状態
- バッジが緑〜グレーで分かりやすく見える
- **訴求点**: 「24h キャッシュで API クォータを節約」

### 5. エクスポート (CSV / コピー) 完了トースト
**キャプチャ対象**: 「CSV」ボタンクリック直後、ダウンロード通知 + トーストが見えるところ
- 画面下にトースト「CSV をダウンロード: …csv」
- **訴求点**: 「結果を CSV / TSV でエクスポート可能」

## 撮影手順 (macOS)

### A. ブラウザサイズを 1280x800 に固定

DevTools のレスポンシブモードで指定するのが楽:
1. ターゲットページを開く
2. F12 で DevTools → デバイスツールバー (`Cmd+Shift+M`)
3. 解像度を **1280 x 800** に手入力 → DPR は 1
4. 「実機表示」のように見えるよう、ツールバーは隠す

または、ウィンドウ自体を 1280x800 にリサイズ:
```bash
# macOS で window をリサイズする AppleScript の例 (Chrome を最前面にしてから実行)
osascript -e 'tell application "Google Chrome" to set the bounds of the front window to {100, 100, 1380, 900}'
```

### B. スクリーンショット

| ショートカット | 範囲 |
|---|---|
| `Cmd + Shift + 4` → スペース → ウィンドウクリック | ウィンドウ単位 (影なし: `Option` 押しながら) |
| `Cmd + Shift + 4` → ドラッグ | 範囲指定 |
| `Cmd + Shift + 5` | スクリーンショット UI (タイマーや動画も可) |

### C. リサイズ・調整

DevTools のレスポンシブモードで撮ったスクリーンショットは DPR 1 なら 1280x800 でそのまま使えます。
通常スクリーンショットを 1280x800 に縮める場合は `sips` で:

```bash
# ファイルを直接 1280x800 にフィット (アスペクト比保持・パディングなし)
sips -z 800 1280 input.png --out output.png

# またはアスペクト比を維持して長辺だけ揃える
sips -Z 1280 input.png --out output.png
```

### D. ファイル名の推奨

```
docs/screenshots/01_popup.png
docs/screenshots/02_results_top.png
docs/screenshots/03_sort_by_likes.png
docs/screenshots/04_cache_badge.png
docs/screenshots/05_export.png
```

## 撮影時のコツ

- **チャンネルは公の有名アカウント** を使うと審査時の印象が良い (例: 自分のチャンネル / Google Developers / TED)
- **API キーは画面に映さない** (オプション画面のキャプチャは不要)
- **個人情報・他人のメールアドレスが映り込まない** ようウィンドウ整理
- **ダークモード / ライトモードどちらかに統一** (ブランド一貫性)
- 1 枚目は **拡張の主訴求** が一目で分かる構図にする (ストア一覧のサムネで使われる)
