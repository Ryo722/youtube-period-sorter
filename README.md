# YouTube 期間別人気動画ソーター

YouTube チャンネルの動画を **指定した期間で絞り込み、再生回数の多い順に並べて表示** する Chrome 拡張機能。

公式 UI には「期間 × 人気順」の組み合わせ絞り込みがないため、これを補完するのが目的。

**[Chrome ウェブストアからインストール](https://chromewebstore.google.com/detail/youtube-%E6%9C%9F%E9%96%93%E5%88%A5%E4%BA%BA%E6%B0%97%E5%8B%95%E7%94%BB%E3%82%BD%E3%83%BC%E3%82%BF%E3%83%BC/gcoblkekjbplafeafmdgcghlcnenfdfp)**

---

## 構成

```
manifest.json              # MV3
popup/                     # 拡張アイコンをクリックしたときの小ウィンドウ
options/                   # API キー設定画面
results/                   # 結果表示用の新規タブページ
background/service-worker.js   # YouTube Data API v3 呼び出し
lib/youtube-api.js         # API ラッパー
```

## インストール

### Chrome ウェブストア (推奨)

[ストアページからインストール](https://chromewebstore.google.com/detail/youtube-%E6%9C%9F%E9%96%93%E5%88%A5%E4%BA%BA%E6%B0%97%E5%8B%95%E7%94%BB%E3%82%BD%E3%83%BC%E3%82%BF%E3%83%BC/gcoblkekjbplafeafmdgcghlcnenfdfp)

### 開発版 (未パッケージ拡張として読み込む)

1. Chrome で `chrome://extensions/` を開く
2. 右上の **「デベロッパーモード」** を ON
3. **「パッケージ化されていない拡張機能を読み込む」** をクリック
4. このディレクトリ (`youtube-period-sorter/`) を選択

## API キーの取得手順

1. [Google Cloud Console](https://console.cloud.google.com/) にアクセスし、プロジェクトを作成 (既存プロジェクトでも可)
2. **「APIとサービス」→「ライブラリ」** で **YouTube Data API v3** を検索 → 有効化
3. **「APIとサービス」→「認証情報」→「認証情報を作成」→「APIキー」**
4. **キー作成画面の入力項目** (画面で求められる順):

   | 項目 | 設定値 |
   |---|---|
   | 名前 | 任意 (例: `youtubeSort`) |
   | このキーを使用してアクセスできる API | **必須** ─「キーを制限」を選択し、一覧から **YouTube Data API v3** のみにチェック |
   | サービス アカウントを介して API 呼び出しを認証する | **オフのまま** (Vertex AI / Gemini 用なので不要) |
   | アプリケーションの制限 | **「なし」のまま**でまず動作確認を推奨。安定運用後に「ウェブサイト」を選んで `chrome-extension://<拡張ID>/*` を追加するとさらに安全 |

5. 発行されたキーをコピー
6. 拡張のアイコンを右クリック → **「オプション」** で API キーを貼り付けて保存
7. **「テスト」** ボタンで疎通確認

### アプリケーションの制限を後から強化する場合

1. `chrome://extensions/` で拡張の **ID** を確認 (例: `abcdefghijklmnopqrstuvwxyz123456`)
2. Google Cloud Console → 認証情報 → 該当キーを編集
3. アプリケーションの制限で「ウェブサイト」を選択
4. ウェブサイトの制限に `chrome-extension://abcdefghijklmnopqrstuvwxyz123456/*` を追加して保存

> **注意**: 開発時の未パッケージ拡張は再読み込みでもIDは安定しますが、別マシン/別フォルダで読み込むと別IDになります。Chrome ウェブストアで公開した場合はストアIDで固定されます。

### クォータについて

- 無料枠: **10,000 ユニット/日**
- 50 件取得: 約 **101 ユニット** (search 100 + videos.list 1)
- 100 件取得: 約 **202 ユニット** (search 200 + videos.list 2)
- **24 時間キャッシュ** が有効なので、同じチャンネル × 期間 × 件数の繰り返しはクォータを消費しません

## 使い方

1. YouTube のチャンネルページ (例: `https://www.youtube.com/@mkbhd`) を開く
2. 拡張アイコンをクリック → チャンネルが自動入力される
3. 期間を選んで「人気順で表示」
4. 新規タブで結果が表示される
5. 結果ページのヘッダーで **期間を切り替えて再取得** も可能

チャンネル指定は以下に対応:
- `@handle` (例: `@mkbhd`)
- `UC` で始まるチャンネル ID (例: `UCBJycsmduvYEL83R_U4JriQ`)

## 機能

- 期間プリセット: 過去 24h / 1週 / 1か月 / 3か月 / 6か月 / 1年 / 全期間
- 件数: 最大 **100 件** (50 / 100 から選択)
- **並び順切替**: 再生回数 / 高評価 / コメント数 / 投稿日 (新しい順) — クライアント側のみ、API は再呼び出ししない
- **24 時間ローカルキャッシュ** (最大 30 エントリ): 同じ条件の再検索はクォータを消費しない
- **強制再取得ボタン**: キャッシュを無視して最新値を取得
- **エクスポート**: CSV ダウンロード (BOM 付き UTF-8、Excel で文字化けしない) / TSV クリップボードコピー (Google スプレッドシートに直接貼り付け可)
- 結果ページのヘッダーで期間・件数・並び順の切り替えが可能
- オプション画面からキャッシュ全削除可能

## 既知の制約 / トレードオフ

- **検索 API の特性**: `search.list` は古い動画ほど取りこぼすことがある (公式仕様)
- **件数上限**: 100 件 (内部で 50 件 × 2 ページ取得)。さらに増やすにはコード側で上限を緩める必要あり
- **再生回数の精度**: API の `viewCount` は若干の遅延あり
- **`/c/` や `/user/` 形式の旧 URL は未対応** (handle か UC ID で指定してください)
